프로그래밍은 변수를 통해 값을 저장하고, 참조하여 연산자로 값을 연산, 평가하고 조건문과 반복문에 의한 흐음제어로 데이터의 흐름을 제어하고
          함수로 대사용이 가능한 구문의 집합을 만들며, 객체, 배열 등으로 자연을 구조화하는 것이다. 

          변수는 값의 위치를 기억하는 저장소이다. 값의 위치란 값이 위치하고 있는 메모리 상의 주소를 의미한다.
          
 즉, 변수란 값이 위치하고 있는 메모리 주소에 
          접근하기 위해 사람이 이해할 수 있는 언어로 명명한 식별자이다. 


          변수 선언 전에는 메모리에 들어가 있지 않지만, var num으로 변수 선언을 하면 000001f위치에 num 들어가지만, 데이터타입은 undifined. 
          값을 할당하면 num =10; 0003a610애 num 이름과 10이 할당된다. num =1000; 로 재할당 된다면  0003a610에 10은 이미 할당되어있고, 
          num이름값과 1000 값은 b92fa017애 할당된다.

        문자열 타입은 텍스트 데이터를 나타내는데 사용한다. 문자열은 0개 이상의 16bit 유니코드 문자들의 집합으로 대부분의 전세계의 문자를
        표현할 수 있다. 문자열은 작은 따옴표 또는 큰 따옴표 안에 텍스트를 넣어 생성한다. 가장 일반적인 표현법은 작은 따옴표를 사용하는 것이다. 

        문자열은 배열처럼 인덱스를 통해 접근할 수 있다. 이와 같은 특성을 갖는 데이터를 유사 배열이라 한다. str[0] = 'S'처럼 이미 생성된 문자열의
        일부 문자를 변경해도 반영되지 않는다. 그런데 에러는 발생하지 않는다. 
        즉 한번 생성된 문자열 read only로서 변경할 수 없다. 이것을 변경 불가능이라고 한다. 

        그러나 새로운 문자열을 재할당하는 것은 물론 가능하다. 이는 기존 문자열을 변경하는 것이 아니라 새로운 문자열을 새롭게 할당하는 것이기 때문이다. 
    
        boolean
        불리언 타입의 값은 논리적 참, 거짓을 나타내는 true 와 false뿐이다. 
        불리언 타입의 값은 참과 거짓으로 구분되는 조건에 의해 프로그램의 흐름을 제어하는 조건문에서 자주 사용한다. 
        비어있는 문자열 null, undifined, 숫자 0는 false로 간주된다. 
    
        undifined타입의 값은 undifined가 유일하다. 
        
선언 이후 값을 할당하지 않은 변수 undifined값을 가진다. 

        즉, 선언은 되었지만 값을 할당하지 않는 변수에 접근하거나 존재하지 않는 객체 프로퍼티에 접근할 경우 undifined가 반환된다. 

        이는 변수 선언에 의해 확보된 메모리 공간을 처음 할당이 이루어질 때까지 빈상태(대부분 비어있지 않고 쓰레기 값이 들어있다.)로 내버려두지 않고, 
        자바스크립트 엔진이 undifined로 초기화하기 때문이다. 

        undifined는 개발자가 의도적으로 할당한 값이 아니라 자바스크립트 엔진에 의해 초기화된 값이다. 변수를 참조했을때 undifined 가 반환된다면,
        참조한 변수가 선언 이후 값이 할당된 적이 없는 변수라는 것을 개발자는 간파할 수 있다. 

        개발자가 의도적으로 undifined를 할당해야하는 경우가 있을까? 자바스크립트 엔진이 변수 초기화에 사용하는 이 값을 만약 개발자가 
        마음대로 할당한다면 undifined의 본래의 취지와 벗어나고, 혼란을 야기함. 그러므로, 변수 값이 없다고 명시할때는 null을 사용.
